# Quick Start Guide

Step-by-step instructions to get the Web Push Notification application running.

## Option 1: Local Development (Recommended for Testing)

### Step 1: Prerequisites

Ensure you have installed:
- Python 3.8+ (Python 3.11 recommended)
- Node.js 18+
- MongoDB 4.4+ (or use Docker for MongoDB)

### Step 2: Start MongoDB

```bash
# Linux (systemd)
sudo systemctl start mongod

# Or manually
mongod

# macOS (Homebrew)
brew services start mongodb-community

# Verify MongoDB is running
mongosh --eval "db.version()"

# Or if mongosh is not available, use mongo:
mongo --eval "db.version()"
```

### Step 3: Backend Setup

```bash
# Navigate to backend directory
cd backend

# Create virtual environment
python3 -m venv venv

# Activate virtual environment
# Linux/macOS:
source venv/bin/activate
# Windows:
# venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

### Step 4: Configure Backend Environment

**Note:** VAPID keys will be automatically generated on first startup if not provided. You can skip manual key generation for quick testing.

**Option A: Let the application auto-generate keys (Easiest)**
- Just start the backend server - keys will be generated automatically on first run
- Keys will be written to `backend/.env` file if the directory is writable

**Option B: Manually generate VAPID keys (Recommended for production)**
```bash
# Make sure you're in backend directory with venv activated
python3 scripts/init_vapid_keys.py

# This will generate keys and write them to backend/.env file automatically
# You can also specify an email:
python3 scripts/init_vapid_keys.py --email mailto:your-email@example.com
```

**Optional: Create .env file manually**
If you want to set MongoDB URI explicitly, create `backend/.env`:
```bash
# Create .env file in backend directory
cat > backend/.env << EOF
MONGODB_URI=mongodb://localhost:27017/push_db
DATABASE_NAME=push_db
VAPID_PUBLIC_KEY=your_public_key_here
VAPID_PRIVATE_KEY=your_private_key_here
VAPID_EMAIL=mailto:your-email@example.com
EOF
```

### Step 5: Create Admin User (Optional but Recommended)

Before accessing the admin portal, create an admin user:

```bash
# Make sure you're in backend directory with venv activated
# MongoDB must be running
python3 scripts/create_admin.py <username> <password>

# Example:
python3 scripts/create_admin.py admin mySecurePassword123
```

**Note:** Password must be at least 8 characters long.

### Step 6: Start Backend Server

```bash
# Make sure you're in backend directory with venv activated
uvicorn main:app --reload

# Backend will automatically:
# - Generate VAPID keys if not found (and write to .env file)
# - Validate existing keys
# - Initialize database connection
# - Start on http://localhost:8000

# Verify by opening http://localhost:8000 in browser
# You should see: {"message": "Web Push Notification Service"}
# Check terminal logs for VAPID key generation status
```

**Keep this terminal open!**

### Step 7: Frontend Setup

Open a **new terminal** window:

```bash
# Navigate to frontend directory
cd frontend

# Install dependencies
npm install

# Create .env file (optional - for local development)
# If not set, frontend will use relative paths which work with nginx proxy in Docker
cat > .env << EOF
VITE_API_BASE_URL=http://localhost:8000
EOF
```

### Step 8: Start Frontend Server

```bash
# Make sure you're in frontend directory
npm run dev

# Frontend should start on http://localhost:3000
# It will automatically open in your browser
```

### Step 9: Test the Application

1. **Open the application** in your browser (http://localhost:3000)
2. **Click "Request Permission"** - Allow notifications when prompted
3. **Enter a User ID** (optional, e.g., "user123")
4. **Click "Subscribe"** - Status should change to "Subscribed"
5. **Enter notification title and body**
6. **Click "Send to User"** or "Broadcast to All"
7. **Check for push notification** on your device

### Step 10: Access Admin Portal (Optional)

1. **Navigate to admin portal**: http://localhost:3000/admin
2. **Login** with the admin credentials you created in Step 5
3. **Manage applications** and view user subscriptions

## Option 2: Docker Deployment (Recommended for Production)

### Step 1: Prerequisites

Ensure you have installed:
- Docker 20.10+
- Docker Compose 2.0+ (or `docker compose` command)

### Step 2: Configure Environment (Optional)

**Note:** VAPID keys will be automatically generated by the Docker entrypoint script if not provided. You can skip this step for quick testing.

**Option A: Let Docker auto-generate keys (Easiest)**
- Just run `docker compose up` - keys will be generated automatically
- Keys will be written to `backend/.env` in the container

**Option B: Pre-generate keys (Recommended for production)**
```bash
# Generate keys before starting Docker
cd backend
python3 -m venv venv
source venv/bin/activate  # Linux/macOS
# On Windows: venv\Scripts\activate
pip install -r requirements.txt
python3 scripts/init_vapid_keys.py

# Keys will be written to backend/.env
# Docker will use these keys from the mounted volume
```

**Option C: Set keys via environment variables**
Create a `.env` file in the project root:
```bash
# Create .env file in project root
cat > .env << EOF
VAPID_PUBLIC_KEY=your_public_key_here
VAPID_PRIVATE_KEY=your_private_key_here
VAPID_EMAIL=mailto:your-email@example.com
EOF
```

### Step 3: Build and Start All Services

```bash
# From project root directory
docker compose up -d --build

# This will:
# - Build backend Docker image (with VAPID key automation)
# - Build frontend Docker image
# - Start MongoDB container
# - Start backend container (auto-generates VAPID keys if needed)
# - Start frontend container

# Check backend logs to see VAPID key generation status:
docker compose logs backend | grep -i vapid

# Or view all logs:
docker compose logs -f
```

### Step 4: Create Admin User (Optional but Recommended)

After services are running, create an admin user:

```bash
# Execute the create_admin script in the backend container
docker compose exec backend python scripts/create_admin.py <username> <password>

# Example:
docker compose exec backend python scripts/create_admin.py admin mySecurePassword123
```

### Step 5: Verify Services Are Running

```bash
# Check service status
docker compose ps

# You should see all three services in "Up" status:
# - push-mongodb (healthy)
# - push-backend (healthy)
# - push-frontend (healthy)
```

### Step 6: Access the Application

- **Frontend:** http://localhost:3000
- **Backend API:** http://localhost:8000
- **MongoDB:** localhost:27017
- **Admin Portal:** http://localhost:3000/admin

### Step 7: Test the Application

1. **Open the application** in your browser (http://localhost:3000)
2. **Click "Request Permission"** - Allow notifications when prompted
3. **Enter a User ID** (optional, e.g., "user123")
4. **Click "Subscribe"** - Status should change to "Subscribed"
5. **Enter notification title and body**
6. **Click "Send to User"** or "Broadcast to All"
7. **Check for push notification** on your device

### Step 8: Access Admin Portal (Optional)

1. **Navigate to admin portal**: http://localhost:3000/admin
2. **Login** with the admin credentials you created in Step 4
3. **Manage applications** and view user subscriptions

### Useful Docker Commands

```bash
# View logs
docker compose logs -f

# View logs for specific service
docker compose logs -f backend
docker compose logs -f frontend
docker compose logs -f mongodb

# Stop all services
docker compose down

# Stop and remove volumes (⚠️ deletes MongoDB data)
docker compose down -v

# Restart a specific service
docker compose restart backend

# Rebuild and restart
docker compose up -d --build
```

## Troubleshooting

### Backend Issues

**Port 8000 already in use:**
```bash
# Find process using port 8000
lsof -i :8000  # macOS/Linux
netstat -ano | findstr :8000  # Windows

# Kill the process or change port in docker-compose.yml
# Note: Use 'docker compose' (v2) or 'docker-compose' (v1) depending on your installation
```

**MongoDB connection error:**
```bash
# Verify MongoDB is running
mongosh --eval "db.version()"

# Check MongoDB logs (if using Docker)
docker compose logs mongodb
```

**VAPID keys not working:**
```bash
# Option 1: Validate existing keys
cd backend
source venv/bin/activate  # Activate venv first
python3 scripts/validate_vapid_keys.py

# Option 2: Regenerate keys manually
python3 scripts/init_vapid_keys.py --force

# Option 3: Let application auto-generate (restart backend)
# Keys will be generated automatically on startup if missing

# Verify .env file has correct keys (no quotes, no extra spaces)
# Check backend logs for key generation/validation messages

# For Docker:
docker compose exec backend python scripts/validate_vapid_keys.py
docker compose exec backend python scripts/init_vapid_keys.py --force
```

### Frontend Issues

**Port 3000 already in use:**
```bash
# Vite will automatically use next available port
# Check terminal output for actual port
```

**Can't connect to backend:**
```bash
# Verify backend is running
curl http://localhost:8000
# Should return: {"message": "Web Push Notification Service"}

# Check CORS settings in backend/main.py
# Check VITE_API_BASE_URL in frontend/.env (for local development)
# In Docker, frontend uses nginx proxy, so VITE_API_BASE_URL is not needed
```

**Service worker not registering:**
```bash
# Open browser DevTools > Application > Service Workers
# Check for errors in Console tab
# Try hard refresh: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (macOS)
```

### Docker Issues

**Services won't start:**
```bash
# Check Docker daemon is running
docker ps

# Check logs for errors
docker compose logs

# Rebuild without cache
docker compose build --no-cache
docker compose up -d
```

**Backend can't connect to MongoDB:**
```bash
# Verify MongoDB is healthy
docker compose ps

# Test network connectivity
docker compose exec backend ping mongodb

# Check MongoDB logs
docker compose logs mongodb

# Verify MongoDB is accessible
docker compose exec mongodb mongosh --eval "db.version()"

# Check backend connection string in docker-compose.yml
# Should be: mongodb://mongodb:27017/push_db
```

**SSL Certificate errors during Docker build:**
```bash
# If you see SSL certificate errors when building (e.g., "hostname doesn't match"),
# this usually means you're behind a proxy that intercepts SSL connections.
# The Dockerfile already includes --trusted-host flags to handle this.

# If the issue persists, try:
# 1. Rebuild without cache
docker compose build --no-cache backend

# 2. Check if you're behind a corporate proxy
# Set proxy environment variables if needed:
# export HTTP_PROXY=http://proxy.example.com:8080
# export HTTPS_PROXY=http://proxy.example.com:8080

# 3. For production, configure proper CA certificates
# Contact your network administrator for the CA certificate bundle
```

## VAPID Key Management

The application includes automated VAPID key generation and management:

### Automatic Key Generation
- **Local Development:** Keys are auto-generated on first startup if not found
- **Docker:** Keys are auto-generated by entrypoint script if not provided
- Keys are automatically written to `backend/.env` file (if writable)

### Manual Key Management

**Local Development:**
```bash
# Navigate to backend directory
cd backend
source venv/bin/activate  # Activate venv first

# Initialize/regenerate keys
python3 scripts/init_vapid_keys.py

# Validate existing keys
python3 scripts/validate_vapid_keys.py

# Generate keys with custom email
python3 scripts/init_vapid_keys.py --email mailto:your-email@example.com

# Force regeneration (overwrites existing keys)
python3 scripts/init_vapid_keys.py --force
```

**Docker:**
```bash
# Validate keys
docker compose exec backend python scripts/validate_vapid_keys.py

# Regenerate keys
docker compose exec backend python scripts/init_vapid_keys.py --force
```

### Key Storage Priority
1. Environment variables (`VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`)
2. `.env` file in backend directory
3. Auto-generation if neither exists
